MySQL

Introduction to MySQL

MySQL is a widely used open-source Relational Database Management System (RDBMS) owned by Oracle Corporation. It's a cornerstone for many web applications due to its reliability, robust performance, and ease of use.

  • Relational Database Management System (RDBMS): Organizes data into tables with rows and columns, establishing relationships between them.
  • Client-Server Architecture: A MySQL server manages the database, and clients (applications, command-line tools) connect to it to perform operations.
  • Key Features:
    • Scalability: Can handle small to very large datasets and high traffic.
    • Security: Robust user management, access control, and encryption features.
    • High Availability: Supports replication and clustering for continuous operation.
    • Open Source: Free to use and modify under the GNU General Public License.

Data Types

Choosing the correct data type for each column is crucial for efficient storage, data integrity, and query performance.

  • Numeric Types:
    • INT: Standard integer (e.g., `INT`).
    • TINYINT: Very small integer, often used for boolean (0 or 1) (e.g., `TINYINT(1)`).
    • FLOAT, DOUBLE: Floating-point numbers for approximate decimal values.
    • DECIMAL(P,S): Fixed-point numbers for exact decimal values, where P is total digits and S is digits after decimal (e.g., `DECIMAL(10,2)` for currency).
  • String Types:
    • CHAR(M): Fixed-length string, padded with spaces (e.g., `CHAR(10)`).
    • VARCHAR(M): Variable-length string, recommended for most text data (e.g., `VARCHAR(255)`).
    • TEXT: For long text strings (up to 65,535 characters).
    • BLOB: For binary large objects (e.g., images, audio).
  • Date/Time Types:
    • DATE: Stores date in 'YYYY-MM-DD' format.
    • TIME: Stores time in 'HH:MM:SS' format.
    • DATETIME: Stores date and time in 'YYYY-MM-DD HH:MM:SS' format.
    • TIMESTAMP: Stores date and time, automatically updated (e.g., for 'last modified' fields).
    • YEAR: Stores year in 2-digit or 4-digit format.
  • Boolean: BOOL or BOOLEAN are aliases for TINYINT(1).

DDL (Data Definition Language)

DDL commands are used to define, modify, and drop database objects like tables, databases, indexes, and views. They deal with the schema of the database.

  • CREATE DATABASE database_name;: Creates a new database.
  • USE database_name;: Selects the database to work with.
  • CREATE TABLE table_name (...);: Creates a new table with specified columns and constraints.

    Example:

    CREATE TABLE products (
    product_id INT AUTO_INCREMENT PRIMARY KEY,
    product_name VARCHAR(255) NOT NULL UNIQUE,
    price DECIMAL(10, 2) DEFAULT 0.00,
    stock_quantity INT CHECK (stock_quantity >= 0),
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES categories(category_id)
);
    Copied!
  • ALTER TABLE table_name ADD COLUMN column_name datatype;: Modifies an existing table (add/drop columns, change data types, add constraints).

    Example:

    ALTER TABLE products ADD COLUMN description TEXT;
ALTER TABLE products MODIFY COLUMN product_name VARCHAR(150);
    Copied!
  • DROP TABLE table_name;: Deletes an entire table, including its data and structure.
  • TRUNCATE TABLE table_name;: Deletes all rows from a table, but keeps the table structure. It's faster than DELETE for removing all rows.
  • DROP DATABASE database_name;: Deletes an entire database. Use with extreme caution!

DML (Data Manipulation Language)

DML commands are used for managing data within schema objects. They affect the content of the database.

  • INSERT INTO table_name (columns) VALUES (values);: Adds new rows of data into a table.

    Example:

    INSERT INTO employees (first_name, last_name, email, hire_date)
VALUES ('Jane', 'Smith', 'jane.smith@example.com', '2023-01-15');
    Copied!
  • SELECT columns FROM table_name WHERE condition;: Retrieves data from one or more tables.

    Example:

    SELECT first_name, last_name FROM employees WHERE hire_date > '2023-01-01';
SELECT DISTINCT department FROM employees; -- Selects unique department names
SELECT product_name AS ItemName, price FROM products; -- Aliasing a column
    Copied!
  • UPDATE table_name SET column1 = value1 WHERE condition;: Modifies existing data in a table.

    Example:

    UPDATE employees SET email = 'john.d@example.com' WHERE employee_id = 101;
UPDATE products SET price = price * 1.10 WHERE category_id = 5;
    Copied!
  • DELETE FROM table_name WHERE condition;: Removes rows from a table.

    Example:

    DELETE FROM employees WHERE status = 'inactive';
DELETE FROM orders WHERE order_date < '2023-01-01';
    Copied!

Clauses

Clauses are used to specify conditions, order, and group results in SQL queries.

  • WHERE: Filters records based on a specified condition.

    Example: SELECT * FROM products WHERE price > 50;

    Operators: =, !=, >, <, >=, <=, AND, OR, NOT, LIKE, IN, BETWEEN, IS NULL.

  • ORDER BY: Sorts the result-set in ascending (ASC) or descending (DESC) order.

    Example: SELECT name, age FROM students ORDER BY age DESC;

  • GROUP BY: Groups rows that have the same values in specified columns into summary rows, often used with aggregate functions.

    Example: SELECT department, COUNT(*) FROM employees GROUP BY department;

  • HAVING: Filters groups based on a specified condition, used after GROUP BY.

    Example: SELECT department, COUNT(*) FROM employees GROUP BY department HAVING COUNT(*) > 5;

  • LIMIT: Restricts the number of rows returned by a query.

    Example: SELECT * FROM products LIMIT 10;

  • OFFSET: Used with LIMIT to skip a specified number of rows before beginning to return rows.

    Example: SELECT * FROM products LIMIT 10 OFFSET 20; (skips first 20, returns next 10)

Joins

Joins are used to combine rows from two or more tables based on a related column between them.

  • INNER JOIN: Returns rows when there is a match in both tables.

    Example: SELECT o.order_id, c.customer_name FROM orders o INNER JOIN customers c ON o.customer_id = c.customer_id;

  • LEFT JOIN (or LEFT OUTER JOIN): Returns all rows from the left table, and the matching rows from the right table. If no match, NULLs are returned for right table columns.

    Example: SELECT c.customer_name, o.order_id FROM customers c LEFT JOIN orders o ON c.customer_id = o.customer_id;

  • RIGHT JOIN (or RIGHT OUTER JOIN): Returns all rows from the right table, and the matching rows from the left table. If no match, NULLs are returned for left table columns.

    Example: SELECT p.product_name, s.supplier_name FROM products p RIGHT JOIN suppliers s ON p.supplier_id = s.supplier_id;

  • FULL OUTER JOIN (emulated in MySQL): Returns all rows when there is a match in one of the tables. MySQL doesn't have a direct FULL OUTER JOIN. It's typically emulated using LEFT JOIN UNION ALL RIGHT JOIN.
  • SELF JOIN: A table is joined with itself. Useful for comparing rows within the same table.

    Example: SELECT a.employee_name AS Employee, b.employee_name AS Manager FROM employees a JOIN employees b ON a.manager_id = b.employee_id;

Functions

MySQL provides many built-in functions to perform calculations, manipulate strings, handle dates, and more.

  • Aggregate Functions: Perform calculations on a set of rows and return a single summary value.
    • COUNT(): Returns the number of rows. (e.g., SELECT COUNT(*) FROM users;)
    • SUM(): Calculates the sum of a numeric column. (e.g., SELECT SUM(amount) FROM orders;)
    • AVG(): Calculates the average value of a numeric column. (e.g., SELECT AVG(price) FROM products;)
    • MIN(): Returns the minimum value in a column. (e.g., SELECT MIN(salary) FROM employees;)
    • MAX(): Returns the maximum value in a column. (e.g., SELECT MAX(order_date) FROM orders;)
  • String Functions: Manipulate string data.
    • CONCAT(s1, s2, ...): Concatenates two or more strings. (e.g., SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM employees;)
    • LENGTH(s): Returns the length of a string in bytes. (e.g., SELECT LENGTH(username) FROM users;)
    • SUBSTRING(s, start, length): Extracts a substring. (e.g., SELECT SUBSTRING(email, 1, 5) FROM users;)
    • UPPER(s) / LOWER(s): Converts string to uppercase/lowercase.
    • TRIM(s): Removes leading/trailing spaces.
  • Date/Time Functions: Work with date and time values.
    • NOW(): Returns the current date and time.
    • CURDATE(): Returns the current date.
    • DATE_FORMAT(date, format): Formats a date value. (e.g., SELECT DATE_FORMAT(created_at, '%Y-%m-%d') FROM users;)
    • DATEDIFF(date1, date2): Returns the number of days between two dates.

Indexes

Indexes are special lookup tables that the database search engine can use to speed up data retrieval. They are similar to an index in a book.

While indexes improve read performance, they can slow down write operations (INSERT, UPDATE, DELETE) as the index also needs to be updated.

  • Types of Indexes:
    • PRIMARY KEY: Uniquely identifies each record in a table; automatically indexed.
    • UNIQUE INDEX: Ensures all values in the column(s) are unique.
    • NORMAL INDEX: Improves search speed on specific columns.
    • FULLTEXT INDEX: For full-text searches on text columns.
  • Creating an Index:
CREATE INDEX idx_employee_lastname ON employees (last_name);
CREATE UNIQUE INDEX idx_user_email ON users (email);
CREATE INDEX idx_product_category_price ON products (category_id, price); -- Composite index
Copied!
  • Dropping an Index:
DROP INDEX idx_employee_lastname ON employees;

Transactions

A transaction is a single logical unit of work. It's a sequence of operations performed as a single logical unit of work. Transactions ensure data integrity by adhering to ACID properties.

  • ACID Properties:
    • Atomicity: All operations within a transaction either complete successfully (commit) or none do (rollback). It's an "all or nothing" principle.
    • Consistency: A transaction brings the database from one valid state to another, maintaining all defined rules and constraints.
    • Isolation: Concurrent transactions execute independently without interfering with each other. The result of concurrent transactions is the same as if they were executed serially.
    • Durability: Once a transaction is committed, its changes are permanent and survive system failures.
  • Transaction Commands:
START TRANSACTION; -- or BEGIN;
-- SQL statements (e.g., INSERT, UPDATE, DELETE)
INSERT INTO accounts (account_id, balance) VALUES (1, 1000);
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

COMMIT; -- Saves all changes made during the transaction permanently
-- OR
ROLLBACK; -- Undoes all changes made during the transaction
Copied!

Best Practices

  • Normalize your database design: Organize tables to reduce data redundancy and improve data integrity. Follow normalization forms (1NF, 2NF, 3NF, BCNF).
  • Use appropriate data types for columns: Select the smallest data type that can hold the required data to save space and improve performance.
  • Write clear, readable, and well-formatted SQL: Use consistent capitalization, indentation, and comments to make your queries understandable.
  • Optimize queries:
    • Use EXPLAIN to analyze query execution plans and identify bottlenecks.
    • Avoid SELECT * in production; explicitly list the columns you need.
    • Use WHERE clauses effectively to filter data early.
    • Consider indexing frequently queried columns.
  • Secure your database:
    • Use strong, unique passwords for database users.
    • Grant minimal necessary privileges to users (principle of least privilege).
    • Regularly audit user accounts and permissions.
    • Keep your MySQL server software updated.
  • Regularly backup your data: Implement a robust backup strategy to prevent data loss.
  • Monitor database performance: Use tools to track query performance, resource utilization, and potential issues.
  • Sanitize and validate user input: Prevent SQL injection attacks by using prepared statements or parameterized queries.
