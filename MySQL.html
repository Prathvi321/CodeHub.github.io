MySQL Notes

DB

Introduction to MySQL

MySQL is a widely used open-source Relational Database Management System (RDBMS).

  • Relational Database Management System (RDBMS)
  • Client-Server Architecture
  • Key Features: Scalability, Security, High Availability
T

Data Types

Common MySQL data types:

  • Numeric: INT, TINYINT, FLOAT, DOUBLE, DECIMAL
  • String: CHAR, VARCHAR, TEXT, BLOB
  • Date/Time: DATE, TIME, DATETIME, TIMESTAMP, YEAR
  • Boolean: BOOL or BOOLEAN (usually TINYINT(1))
DDL

DDL (Data Definition Language)

Commands for defining and managing database structures:

  • CREATE DATABASE: Creates a new database.
  • USE DATABASE: Selects a database.
  • CREATE TABLE: Creates a new table.
  • ALTER TABLE: Modifies an existing table.
  • DROP TABLE: Deletes a table.
  • DROP DATABASE: Deletes a database.
DML

DML (Data Manipulation Language)

Commands for managing data within tables:

  • INSERT INTO: Adds new rows.
  • SELECT: Retrieves data.
  • UPDATE: Modifies existing rows.
  • DELETE: Removes rows.
SQL

Basic Queries

Creating a Database:

CREATE DATABASE mydatabase;

Using a Database:

USE mydatabase;

Creating a Table:

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100) UNIQUE,
    registration_date DATE
);

Inserting Data:

INSERT INTO users (name, email, registration_date)
VALUES ('John Doe', 'john.doe@example.com', '2024-05-28');

Selecting Data:

SELECT * FROM users WHERE name = 'John Doe';
C

Clauses

Keywords to refine queries:

  • WHERE: Filters records.
  • ORDER BY: Sorts results.
  • GROUP BY: Groups rows.
  • HAVING: Filters groups.
  • LIMIT: Restricts row count.
  • OFFSET: Skips rows.
JN

Joins

Combining rows from two or more tables:

  • INNER JOIN
  • LEFT JOIN
  • RIGHT JOIN
  • FULL OUTER JOIN (emulated)
  • SELF JOIN
FN

Functions

Built-in operations:

  • Aggregate: COUNT(), SUM(), AVG(), MIN(), MAX()
  • String: CONCAT(), LENGTH(), SUBSTRING()
  • Date/Time: NOW(), CURDATE(), DATE_FORMAT()
IX

Indexes

Improve query performance by enabling faster data retrieval.

CREATE INDEX idx_lastname ON employees (lastname);
TR

Transactions

Ensure ACID properties (Atomicity, Consistency, Isolation, Durability).

START TRANSACTION;
-- SQL statements
COMMIT; -- or ROLLBACK;
BP

Best Practices

  • Normalize your database design.
  • Use appropriate data types for columns.
  • Write clear, readable, and well-formatted SQL.
  • Optimize queries: use EXPLAIN to analyze.
  • Secure your database: strong passwords, minimal privileges.
  • Regularly backup your data.
  • Avoid SELECT * in production for specific needs.
